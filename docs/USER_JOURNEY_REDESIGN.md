# 漫剧工坊 — 用户旅程重设计方案

> 核心原则：**产品是一条流水线，不是一个仪表盘。**
> 用户的心智模型："我有故事 → AI帮我变成视频 → 我在关键节点把关"

---

## 一、用户旅程总览

```
新用户首次使用：

  项目列表 ──→ 创建向导 ──→ Pipeline控制台 ──→ Gate审核(循环) ──→ 成片导出
    │              │              │                  │
    │         3步完成设定      实时看AI工作         审核/打回/微调
    │
  老用户回来：
    │
    └──→ 点击项目卡片 ──→ Pipeline控制台（自动恢复到上次位置）
```

### 完整旅程分解（7个阶段）

| # | 阶段 | 用户在哪 | 用户做什么 | 系统做什么 |
|---|------|---------|----------|----------|
| 1 | **发现/进入** | 项目列表 | 选择已有项目 / 点"新建" | 展示项目卡片+Pipeline进度摘要 |
| 2 | **项目设定** | 创建向导(3步) | 输入故事+选风格+确认 | 验证输入，创建项目实体 |
| 3 | **启动生产** | Pipeline控制台 | 点"开始生成" | 启动Pipeline，SSE推送进度 |
| 4 | **等待+观察** | Pipeline控制台 | 看AI实时工作进度 | 依次执行各Stage，推送事件 |
| 5 | **Gate审核** | Gate审核页 | 审核AI产出，通过/打回 | 暂停Pipeline，等待决策 |
| 6 | **微调(可选)** | 资产/剧本/分镜编辑器 | 手动修改AI产出细节 | 保存修改，重新生成受影响内容 |
| 7 | **完成/导出** | 成片预览 | 预览最终视频，导出/发布 | 输出最终MP4 |

---

### 阶段 5：Gate审核详情

**路由**: `/projects/:id/gate/:gate`（已有GateReview.tsx）

**布局**: Sidebar（project context）+ 内容区

每个Gate有不同的审核内容：

| Gate | 审核什么 | 用户看到什么 |
|------|---------|----------|
| **gate_creative** | 世界观+角色+场景+道具+剧本+分镜 | 资产卡片网格 + 剧本预览 + 分镜缩略图 |
| **gate_storyboard** | 角色插图+场景插图 | 大图预览，可对比参考描述 |
| **gate_production** | 面板合成图+视频+配音+BGM | 逐面板的图/视频/音频播放器 |
| **gate_final** | 最终合成视频 | 完整视频播放器 + QA报告 |

**关键交互**：
- 可以逐项审核（单个角色通过/打回）
- "退回修改"时可以指定`regeneration_targets`（只重新生成被标记的项）
- "批准通过" → 自动回到Pipeline控制台，Pipeline继续执行下一阶段

### 阶段 6：微调编辑器（从Gate进入）

这是现有的 **资产仓库 / 剧本编辑器 / 分镜编辑器** 页面，但进入方式改变：

**旧方式**：从侧边栏直接点击进入（随时可达）
**新方式**：从Gate审核页的"查看详情并微调"按钮进入

**路由保持不变**：
- `/projects/:id/assets` — 资产微调
- `/projects/:id/episodes/:eid/script` — 剧本微调
- `/projects/:id/episodes/:eid/storyboard` — 分镜微调

**改造**：
- 页面顶部增加"返回审核"导航条
- 编辑完成后"保存并返回审核"按钮
- 这些页面不再是"从零创建"的工具，而是"微调AI产出"的工具

## 三、导航结构重设计

### 侧边栏（Project Context）

**旧结构**（功能模块平铺）：
```
概览
剧集管理
资产仓库
团队协作
```

**新结构**（Pipeline阶段 + 工具入口）：

```
── 项目 ──
  概览（Pipeline控制台）     ← 核心入口

── 阶段 ──
  前期策划     ✓ 完成
  创意开发     ✓ 完成
  资产制作     ● 进行中
  生产制作     ○ 等待
  后期制作     ○ 等待

── 工具 ──
  资产仓库                  ← 仍可直接访问
  剧本编辑器                ← 仍可直接访问
  分镜编辑器                ← 仍可直接访问
  项目设置
```

**设计说明**：
- "阶段"区域展示Pipeline进度，点击已完成的阶段可查看该阶段的产出
- "工具"区域保留直接访问编辑器的能力（高级用户需要）
- 但新用户自然跟随Pipeline流程，不需要理解工具区

### 点击阶段的行为

| 阶段状态 | 点击行为 |
|---------|---------|
| ✓ 已完成 | 查看该阶段的产出物（只读预览） |
| ● 进行中 | 滚动到Pipeline控制台的当前Stage |
| 🔔 等待审核 | 跳转Gate审核页 |
| ○ 等待 | 显示"该阶段尚未开始"提示 |

## 五、关键交互流程（Sequence）

### 流程 A：新用户首次完整体验

```
用户                           系统
 │                              │
 │  打开首页                     │

 │ ───────────────────────────→ │
 │                              │  POST /projects (创建项目)
 │                              │  POST /projects/:id/runs (启动Pipeline)
 │                              │  302 → /projects/:id (Pipeline控制台)
 │                              │
 │  看到Pipeline实时进度          │
```

## 七、空状态处理

| 场景 | 当前设计（错误） | 正确设计 |
|------|---------------|---------|
| 新项目Dashboard | "暂无剧集" + "新建剧集"按钮 | "开始生成"大按钮，一键启动Pipeline |
| 资产仓库空 | "没有符合条件的角色" | 不应该直接进入——Pipeline会自动填充 |
| 剧本编辑器空 | "暂无剧本内容" | 不应该直接进入——Pipeline会自动生成 |
| 分镜编辑器空 | "暂无分镜面板" | 不应该直接进入——Pipeline会自动生成 |
| 项目列表空 | 当前可以 | "创建你的第一个AI漫画项目" CTA + 示例项目 |

**原则**：用户永远不应该看到"空"的编辑器。如果Pipeline还没生成内容，就不要让用户进入编辑器。

## 八、实现优先级

### Phase 1（核心流程可跑通）

1. **创建向导** `/projects/new` — 3步向导，创建完自动启动Pipeline
2. **Pipeline控制台** 改造 `/projects/:id` — 替代当前Dashboard，SSE实时进度
3. **Gate审核** 完善 `/projects/:id/gate/:gate` — 4个Gate的审核UI
4. **侧边栏重构** — 从功能模块 → 阶段进度 + 工具入口

### Phase 2（体验完善）

5. **成片预览页** `/projects/:id/preview` — 视频播放+QA报告+导出
6. **项目卡片** 改造 — 展示Pipeline进度条而非抽象百分比
7. **编辑器"返回审核"** — 微调模式导航条

### Phase 3（细节打磨）

8. 逐项审核（单个角色通过/打回）
9. 候选媒体选择（多个生成结果选最佳）
10. 版本对比（undo/redo历史）
11. 多剧集管理

## 九、需要删除/降级的功能

| 功能 | 处理方式 | 原因 |
|------|---------|------|
| Dashboard上的"新建剧集"按钮 | **删除** | 剧集由Pipeline自动创建 |
| 侧边栏"剧集管理"入口 | **降级到工具区** | 不是主流程的一部分 |
| 侧边栏"团队协作"入口 | **删除** | MVP不需要，增加认知负担 |
| `/settings` 全局设置页 | **合并到项目设置** | Provider配置是per-project的 |
| `/docs` 文档中心 | **删除** | 内联帮助更好 |
| `/help` 帮助中心 | **改为底部外链** | 不需要独立页面 |
| "查看全部"按钮 | **已删除** | 无目标页面 |

## 十、核心度量指标

| 指标 | 含义 | 目标 |
|------|------|------|
| **首次成片时间** | 新用户从注册到拿到第一个视频 | < 15分钟 |
| **Gate通过率** | 用户直接通过（不退回）的Gate比例 | > 70% |
| **退回后重做成功率** | 退回后第二次通过的比例 | > 90% |
| **Pipeline完成率** | 启动的Pipeline最终跑完的比例 | > 80% |
| **单集成本** | 生产一集1.5分钟视频的AI花费 | < $5 |

## 总结

这个产品的用户旅程应该是：

**创建 → 启动 → 观察 → 审核 → 完成**

而不是：

**创建 → 手动建剧集 → 手动写剧本 → 手动排分镜 → 手动管资产 → ...**

AI做重活，人做决策。前端应该围绕这个核心循环来设计。
